# API Testing Project using Postman

## 1. Introduction
This project presents a practical implementation of API testing using Postman. The objective is to evaluate and validate the functionality, reliability, and correctness of RESTful APIs for a Customer Management System.

---

## 2. Project Objectives
- To perform manual API testing using Postman  
- To validate API responses and status codes  
- To test CRUD (Create, Read, Update, Delete) operations  
- To identify potential issues in API behavior  

---

## 3. Scope of Testing
The following APIs were tested as part of this project:

- Create Customer (POST)
- User Login (POST)
- Retrieve Customer/Profile (GET)
- Update Customer Information (PUT/PATCH)
- Delete Customer (DELETE)

---

## 4. Testing Methodology
The testing process included:

- Sending HTTP requests using Postman (GET, POST, PUT, DELETE)  
- Verifying HTTP status codes (200, 201, 400, 404, 500)  
- Validating JSON response structure and data  
- Performing both positive and negative test scenarios  
- Ensuring proper functionality of all endpoints  

---

## 5. Tools and Technologies
- Postman  
- REST API  
- JSON  

---

## 6. Testing Environment
All APIs were tested in a local development environment using the following base URL:
http://localhost:3000

**Note:** The APIs are hosted locally; therefore, they are not accessible externally.

---

## 7. Conclusion
This project demonstrates a foundational understanding of API testing practices and tools. It reflects the ability to perform structured testing, analyze API responses, and ensure application quality through systematic validation.
